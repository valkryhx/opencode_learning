# oh-my-opencode 插件安装

## 查看 npm 配置

```bash
npm config get prefix      # 查看全局安装路径
npm config get cache      # 查看缓存路径
npm root -g               # 查看全局 node_modules 路径
```

## 设置 npm 配置 避免都安装到C盘

```bash
npm config set prefix "G:\nodejs_home\node_global"   # 设置全局安装路径
npm config set cache "G:\nodejs_home\node_cache"      # 设置缓存路径
```

## 安装命令

### 全局安装（推荐）

```bash
npm i -g oh-my-opencode
```

## 配置文件

插件安装后，需要在 `C:\Users\Administrator\.config\opencode\` 目录下创建 `oh-my-opencode.json` 配置文件：

```json
{
  "$schema": "https://raw.githubusercontent.com/code-yeongyu/oh-my-openagent/master/assets/oh-my-opencode.schema.json"
}

也可以将这个github上的文件下载到本地 配置修改成

{
  "$schema": "./oh-my-opencode.schema.json"
}

```
> [!IMPORTANT]
> `opencode.json` 中的 `plugin` 配置必须指向 `"oh-my-opencode"`。

##  常见问题：为何 C 盘还有文件写入？

`oh-my-opencode` 作为一个功能强大的多 Agent 编排框架，会向 `C:\Users\Administrator\.config\opencode\node_modules` 写入包文件。这主要是基于以下**核心架构设计**：

###  1. 插件运行时的“沙箱隔离” (Sandbox Isolation)

OpenCode 的设计理念是将**自身运行环境、插件环境**与**日常开发的业务项目**完全隔离：

*   **不污染项目**：不会将工具安装在当前项目的 `node_modules` 中（保护业务代码和 `package.json`）。
*   **防版本冲突**：不会安装在系统的全局 Node 目录下（避免不同 AI 插件之间的依赖版本冲突）。

**结论：**
`C:\Users\Administrator\.config\opencode\` 是 OpenCode 专属的**独立虚拟工作区 (Workspace)**。所有插件运行所需的代码和依赖，都会被安全地圈定在此目录中。
