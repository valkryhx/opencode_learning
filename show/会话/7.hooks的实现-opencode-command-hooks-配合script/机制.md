# OpenCode Command Hooks 机制

> **前置要求**：需在全局 `opencode.json` 的 `plugins` 中加入 `"opencode-command-hooks"` 插件。

```json
{
  "plugins": ["opencode-command-hooks"]
}
```

## 触发原理

OpenCode 启动时自动读取全局配置目录下的 `command-hooks.jsonc`：

```
C:/Users/Administrator/.config/opencode/command-hooks.jsonc
```

每次 tool call 结束后，自动执行配置中 `run` 字段指定的命令。

## 配置文件结构

```jsonc
{
  "tool": [           // 监听 tool call 事件
    {
      "id": "beep-on-done",
      "when": {
        "phase": "after"   // 每次 tool call 结束后触发
      },
      "run": "python C:/Users/Administrator/.config/opencode/scripts/beep.py"
    }
  ]
}
```

## Scripts 存放位置

所有 hook 调用的脚本统一放在全局配置目录的 `scripts/` 子目录下：

```
C:/Users/Administrator/.config/opencode/
├── command-hooks.jsonc   # hook 配置文件
└── scripts/              # 脚本存放目录
    ├── beep.py           # 响铃脚本
    ├── tianwang.py       # 弹窗触发脚本
    └── tianwang.vbs      # VBScript 弹窗内容
```

`run` 字段中使用绝对路径引用脚本，避免工作目录不确定导致的找不到文件问题。

## Scripts 放置位置

scripts 统一放在全局配置目录下的 `scripts/` 子目录：

```
C:/Users/Administrator/.config/opencode/scripts/
```

该目录需手动创建，opencode 不会自动生成。`command-hooks.jsonc` 中 `run` 字段直接引用此路径下的脚本文件。

---

## 脚本内容

`C:/Users/Administrator/.config/opencode/scripts/beep.py`

```python
import winsound
winsound.Beep(800, 300)  # 800Hz，响 300ms
```

## 触发链路

1. OpenCode 启动时读取全局 `command-hooks.jsonc`
2. 每次 tool call 结束（`phase: "after"`），自动执行 `run` 里的命令
3. `beep.py` 调用 Windows `winsound.Beep(800, 300)` 发出提示音

## 关键字段说明

| 字段              | 说明                              |
| ----------------- | --------------------------------- |
| `"tool"`          | 事件类型，对应 tool call 生命周期 |
| `phase: "before"` | tool call 开始前触发              |
| `phase: "after"`  | tool call 结束后触发              |
| `run`             | 执行的 shell 命令，支持任意脚本   |

## 生效方式

配置文件放在全局配置目录 `~/.config/opencode/` 下即自动生效，**无需重启，热加载**。

## 进阶：按 turn 触发（而非每次 tool call）

`tool` 事件在每次 tool call 结束后都触发，一轮对话可能响几十次。  
改用 `session` + `event: "session.idle"` 可以做到每轮 assistant 响应完成后只触发一次。

```jsonc
{
  // 每轮对话结束后响铃
  "session": [
    {
      "id": "beep-on-turn-end",
      "when": {
        "event": "session.idle"
      },
      "run": "python C:/Users/Administrator/.config/opencode/scripts/beep.py"
    }
  ]
}
```

`session.idle` 含义：assistant 完成本轮响应、session 进入等待用户输入状态时触发。

## 事件类型对比

| 事件类型                            | 触发时机                | 频率             |
| ----------------------------------- | ----------------------- | ---------------- |
| `tool` + `phase: "after"`           | 每次 tool call 结束     | 高（每轮几十次） |
| `session` + `event: "session.idle"` | 每轮 assistant 响应完成 | 低（每轮一次）   |

## Hook 输出限制与绕过方案

### 核心限制

opencode 的 hook 执行环境中：

- **stdout** → 被捕获送给 AI 作为 context，用户看不到
- **stderr** → 被静默丢弃，用户也看不到

这是已知的设计限制（参考 Claude Code issue #33674），**没有任何 stdout/stderr 方式能让用户在 terminal 看到 hook 输出**。

beep 能正常工作是因为它调用系统音频 API，不依赖 stdout/stderr。

### 绕过方案：Windows 弹窗通知

通过 VBScript + `ChrW()` 实现 Unicode 弹窗，绕开所有编码限制：

**`tianwang.vbs`**（ASCII 编码保存）：

```vbscript
MsgBox ChrW(22825)&ChrW(29579)&ChrW(30422)&ChrW(22320)&ChrW(34382), 0, "opencode"
```

> `ChrW()` 接受 Unicode code point（十进制），避开 GBK/UTF-8 编码问题。
> 天=22825 王=29579 盖=30422 地=22320 虎=34382

**`tianwang.py`**（hook 调用入口）：

```python
import subprocess
subprocess.Popen(
    ['wscript', r'C:/Users/Administrator/.config/opencode/scripts/tianwang.vbs'],
    stdout=subprocess.DEVNULL,
    stderr=subprocess.DEVNULL
)
```

> 用 `Popen`（非阻塞）确保 hook 不卡住 opencode 主进程。

### 生成 VBScript 的方式

直接写中文到 vbs 文件会有编码问题，用 Python 生成：

```python
chars = [ord(c) for c in '天王盖地虎']
parts = '&'.join(f'ChrW({c})' for c in chars)
line = f'MsgBox {parts}, 0, "opencode"'
with open('tianwang.vbs', 'w', encoding='ascii') as f:
    f.write(line + '\n')
```

### 踩坑记录

| 方案                                     | 问题                             |
| ---------------------------------------- | -------------------------------- |
| `print()` / `print(file=sys.stderr)`     | hook 环境下完全不可见            |
| `NotifyIcon.ShowBalloonTip` (PowerShell) | Win11 已弃用，无消息循环不渲染   |
| WinRT Toast API                          | 需要注册 AppId，直接调用无效     |
| `msg %USERNAME%`                         | 远程会话环境下报错               |
| `mshta vbscript:msgbox(...)`             | bash 引号嵌套导致命令解析失败    |
| VBScript UTF-8 编码                      | wscript 报「无效字符」编译错误   |
| VBScript GBK 编码                        | wscript 报「语句未结束」编译错误 |
| **VBScript ASCII + `ChrW()`**            | **正常工作**                     |
