# OpenCode 与智能体开发实操作业

共 9 题，满分 100 分。每题完成后保存 opencode 对话的 share 链接作为提交凭证。
大部分题目需要提交opencode session记录 ，使用方式如下：
进入opencode执行：
```
/export 
```
会导出本会话的markdown格式记录文件，修改filename为作业题目编号，例如：作业1.md，推荐将该markdown文件作为提交内容。

如果需要切换会话，可以进入opencode执行：
```
/sessions
```
然后选择之前完成作业的会话，执行 /export 导出markdown文件。

## 题 1：模型配置、插件机制、MCP 配置格式（10 分）

指定一个可用的第三方模型（非默认），添加至少一个插件（如 `oh-my-opencode` 或 `opencode-command-hooks`），并配置一条 MCP server 条目。你可以让opencode自己帮你做，提交session记录。

**考察点**：模型配置、插件机制、MCP 配置格式。

---

## 题 2：编写一个实用 Skill（12 分）

使用 `skill-creator` 创建一个 skill，功能自定。要求：

- skill 描述清晰，能被 opencode 在合适时机自动触发
- 包含至少一个具体的操作步骤或 checklist
- 在一次真实任务中实际调用该 skill，并提供对话seesion记录

**考察点**：skill 的结构设计、触发描述的准确性、实际可用性。

---

## 题 3：搭建并接入 MCP Server（12 分）

用 `fastmcp` 或标准 MCP SDK 编写一个本地 MCP server，暴露至少一个 tool（例如查系统时间、读本地文件目录、查 git log 等）。在 `opencode.json` 中注册该 server，并在 opencode 对话中成功调用该 tool，展示返回结果。
提交session记录。

**考察点**：MCP server 的编写与注册流程、tool 定义格式、opencode 与本地服务的通信。

---

## 题 4：创建并触发 Hook（10 分）

安装 `opencode-command-hooks` 插件，编写 `command-hooks.jsonc`，实现以下任意一种触发效果：

- 每轮对话结束后执行一个脚本（如系统提示音、写日志文件）
- 在特定 tool call 前后执行自定义逻辑

提供 `command-hooks.jsonc` 内容和对应脚本。

**考察点**：hook 配置结构、`phase`/`event` 字段的区别、脚本与 hook 的联动。

---

## 题 5：使用 Playwright Skill 完成浏览器自动化（12 分）

加载 `playwright-cli` skill，让 opencode 控制浏览器完成以下任务：打开一个目标网页，提取页面中的特定信息（如标题、某段正文、表格数据），并将结果写入本地文件。

提供提交session记录，里面可以包含 playwright 操作过程和最终写入的文件内容。

**考察点**：playwright skill 的加载与使用、页面内容提取、结果落地到文件。

---

## 题 6：基于 ripgrep 实现极简 RAG Skill（12 分）

编写一个 skill，利用 `rg`（ripgrep）对本地知识库目录进行全文检索，将检索结果作为上下文注入给 AI，回答用户针对该知识库的问题。不依赖向量数据库，不调用 embedding API。

要求：
- skill 中说明检索流程（接收 query → rg 搜索 → 拼接结果 → AI 回答）
- 实际运行一次，提供 query、检索结果片段和最终回答。提交session记录。

**考察点**：RAG 的核心思路、ripgrep 的实用性、skill 对工具调用流程的封装。

---

## 题 7：将 opencode 作为 HTTP 服务调用（12 分）

以服务模式启动 opencode，编写一个脚本（Python 或 Node.js），通过 HTTP API 向该 opencode 实例发送一条消息并获取回复。

在此基础上，可选实现一个极简的浏览器对话界面（HTML + fetch）。提供服务启动命令、调用脚本代码和运行结果截图。提交session记录。

**考察点**：opencode server 模式的启动参数、API 请求格式、跨进程调用的可行路径。

---

## 题 8：Session 历史检索与整理（10 分）

使用 opencode 内置的 `session_search`、`session_read` 等工具，从本地历史对话中检索一个特定主题（例如某次调试过程、某个配置问题），提取关键信息，整理成一份简短的技术笔记写入本地 `test.md` 文件。提交session记录。

**考察点**：session 管理工具的使用、历史对话的检索与复用价值。

---

## 题 9：GUIDE.md 与全局 Instruction 设计（10 分）

为自己的 opencode 全局配置目录编写一份 `GUIDE.md`（即全局 instruction 文件），内容包括：

- 你常用的编码规范或约定（如语言、注释风格、文件路径约定）
- 至少一条针对你本地环境的特殊说明（如编码设置、工具路径、禁止行为）
- 一条对 AI 行为的明确约束（如禁止提交代码、禁止使用某类 API）

提供最终 `GUIDE.md` 内容。

**考察点**：instruction 的设计思路、对 AI 行为的约束能力、全局配置的实际作用。
